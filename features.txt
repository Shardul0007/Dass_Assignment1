- IIIT domain enforcement for IIIT participants [DONE]

- Non-IIIT participant signup supported [DONE]

3.2 Organizer authentication
- no self-registration; created by admin [DONE]
- login via admin-provisioned creds [DONE]
- organizer password reset must be requested and handled by admin [DONE]
  - organizer sends request via /profile Security section
  - admin approves/rejects in Password Reset Requests page

3.3 Admin provisioning
- Admin is the first user; backend-only provisioning/seed [MISSING]

3.4 Security requirements
- passwords hashed via bcrypt in backend [DONE]
- JWT auth used on protected routes [DONE]
- role-based access control in frontend routes [DONE]

3.5 Session management
- token stored in localStorage; persists across restarts [DONE]
- logout clears localStorage [DONE]
